Welcome to Slingstanbol

Sling-Stanbol integrates Apache Sling and Apache Stanbol. It provides features to access Stanbol functionality to enhance and process Sling content.

Stanbol

All Stanbol services are available in the OSGi environment and can be accessed by Sling components. The unaltered Stanbol webinterface is available at /stanbol/, it should provide all functionality as in the Stanbol full launcher, post a bug if something is missing.

Sling

Sling-Stanbol is a Sling launcher extending the default Sling Launcher with bundles to integrate the Stanbol features. It also includes support for JAX-RS resources and includes the Clerezza JAX-RS Utilities for RDF.

Sling-Stanbol

Sling-Stanbol provides a Resource-Page that can be used enhance HTML resources with RDFa annotations. Thanks to the Stanbol suggestions annotating documents becomes easy.

You may upload HTML documents via WebDav or check it out with one of the example documents:

  • /slingstanbol/howl.html.stanbol
  • /slingstanbol/turkey.html.stanbol

On the resource-page you an also see pre-computed meta-data, this is generated whenever a document is added or modified. It can be accessed by adding ".rdf" to the end of the URI (webinterface).

The precomputed metdata is stored in a graph than can also be accessed from Stanbol, for instance you can use the Stanbol Sparql endpoint to query it. You can, for example, use the form at /stanbol/sparql and the following query to get all triples from the enhancement graph:

	SELECT *
	WHERE {
	    GRAPH <urn:x-localinstance:/enhancement.graph> {?s ?p ?o}
	}
